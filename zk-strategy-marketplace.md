# zkStrategy: エージェント向け秘匿型ストラテジー・マーケットプレイス

## 一言で言うと

> **トレーディング戦略を秘密にしたまま、その成績が本物であることを証明し、AI エージェントがワンクリックで購入・実行できるマーケットプレイス。**

---

## 解決する課題

### ストラテジー提供者（作る人）の問題

| 課題 | 現状 |
|---|---|
| **公開したら終わり** | いい戦略を見つけても、公開した瞬間にコピーされアルファが消滅する |
| **成績を証明できない** | 「年利 40% 出してます」と言っても、誰も信じない。スクショは改竄できる |
| **マネタイズ手段がない** | 個人が優秀な戦略を持っていても、ファンドに所属しないと収益化できない |

### 投資家（使う人）の問題

| 課題 | 現状 |
|---|---|
| **信頼できない** | 戦略の実績が本物かどうか検証する手段がない |
| **中身が見えない不安** | ブラックボックスに資金を預けることへの抵抗 |
| **手動オペレーション** | 戦略を見つけても、自分で実装・運用する必要がある |

### AI エージェント時代の問題

| 課題 | 現状 |
|---|---|
| **エージェントが戦略を探せない** | OpenClaw 等のエージェントが使える「戦略 API」が存在しない |
| **決済の壁** | エージェントが自律的に戦略を購入・サブスクする仕組みがない |
| **品質の担保** | エージェントがどの戦略を信頼すべきか判断できない |

## コアアイデア：zkTLS による「ゼロ知識ストラテジー証明」

### zkTLS とは

TLS（HTTPS 通信）のレスポンスを**ゼロ知識証明**で検証可能にする技術。

```
通常の流れ：
  ユーザー → Binance API → 「あなたの年間リターンは +42%」

zkTLS の流れ：
  ユーザー → Binance API → 「あなたの年間リターンは +42%」
       ↓
  この TLS レスポンスから ZK 証明を生成
       ↓
  「Binance が返した本物のデータで +42% であること」を
  戦略の中身を明かさずに証明できる
```

### 何が証明できるか

| 証明対象 | データソース（TLS） | 証明内容 |
|---|---|---|
| 取引成績 | 取引所 API（Binance, Coinbase 等） | 「この戦略は過去 1 年で +42% のリターンを出した」 |
| シャープレシオ | 取引所 API | 「リスク調整後リターンが 2.1 以上である」 |
| 最大ドローダウン | 取引所 API | 「最大 DD が -15% 以内に収まっている」 |
| 取引頻度 | 取引所 API | 「月平均 50 回以上取引している（アクティブ）」 |
| SNS 実績 | Twitter/X API | 「このアカウントのフォロワーが 10k 以上」 |
| 本人確認 | 各種 API | 「この人物は実在し、この取引所アカウントを所有している」 |

**ポイント：戦略のロジック（どの銘柄を、いつ、なぜ買うか）は一切公開されない。**

## エージェント統合（OpenClaw プラグイン）

### ユーザー体験

```
ユーザー: 「月利 5% 以上、DD 10% 以下の暗号通貨戦略を探して」

OpenClaw Agent:
  → zkStrategy API を呼び出し
  → zkTLS 証明済みの戦略を検索
  → 上位 3 件を提示（成績証明付き）

ユーザー: 「2 番目の戦略をサブスクして」

OpenClaw Agent:
  → 決済実行（ステーブルコイン / Stripe）
  → シグナルフィードを購読開始
  → 以降、シグナル受信 → 自動売買実行
```

### OpenClaw プラグイン設計

```typescript
// zkStrategy プラグイン
api.registerTool({
  name: "zk_strategy_search",
  description: "zkTLS で成績が証明された戦略を検索する",
  parameters: {
    min_return: { type: "number", description: "最低リターン (%)" },
    max_drawdown: { type: "number", description: "最大DD上限 (%)" },
    asset_class: { enum: ["crypto", "us_equity", "fx"] },
    min_sharpe: { type: "number" }
  }
})

api.registerTool({
  name: "zk_strategy_subscribe",
  description: "戦略をサブスクライブし、シグナルを受信開始する",
  parameters: {
    strategy_id: { type: "string" },
    allocation: { type: "number", description: "配分額 (USD)" }
  }
})

api.registerTool({
  name: "zk_strategy_verify",
  description: "戦略の zkTLS 証明を検証する",
  parameters: {
    strategy_id: { type: "string" },
    proof_type: { enum: ["return", "sharpe", "drawdown", "identity"] }
  }
})

// シグナル受信 → 自動実行（Service として常駐）
api.registerService({
  name: "zk_signal_listener",
  start: async () => {
    // WebSocket でシグナルフィードを購読
    // シグナル受信 → broker_api で自動執行
    // リスク Hook が自動適用される
  }
})
```

## ビジネスモデル

```
Strategy Provider ──シグナル──→ Platform ──シグナル──→ Investor/Agent
                                  │
                        手数料（成果報酬 or 月額）
                                  │
                    ┌─────────────┼─────────────┐
                    │             │             │
                Provider: 70%  Platform: 20%  Protocol: 10%
```

| 収益源 | モデル |
|---|---|
| サブスクリプション | 月額 $XX〜で戦略シグナルを購読 |
| 成果報酬 | 利益の X% をプロバイダーとプラットフォームで分配 |
| 証明手数料 | zkTLS 証明生成ごとに少額手数料 |
| API アクセス | エージェント向け API コール課金 |

## 競合との差別化

| 既存サービス | 問題 | zkStrategy の優位性 |
|---|---|---|
| QuantConnect / Numerai | 戦略を提出する必要がある（秘匿できない） | **戦略ロジック完全秘匿** |
| コピートレード（eToro 等） | 成績の改竄リスク、戦略が丸見え | **zkTLS で成績を暗号学的に証明** |
| Hive5（GKTLS） | 汎用的な zkTLS | **Agentic に特化**、エージェントが直接消費 |
| Signal プロバイダー（Telegram） | 信頼性ゼロ、手動コピー | **証明付き + 自動執行** |

**最大の差別化：「エージェントファースト」**
- 人間が UI でポチポチするのではなく、**AI エージェントが自律的に戦略を評価・購入・実行**する
- OpenClaw のプラグインとして提供 → 既存の 100k+ ユーザーベースにリーチ

## まとめ

```
従来：  戦略を公開 → コピーされる → アルファ消滅
        成績を主張 → 証明できない → 信頼されない
        戦略を発見 → 手動で実装 → 運用が大変

zkStrategy：
        戦略を秘匿 → zkTLS で成績だけ証明 → 信頼を獲得
        エージェントが検索 → 自動サブスク → 自動実行
        プロバイダーは報酬を得る → 持続的なエコシステム
```

> **「優秀な戦略を持つ個人が、秘密を守りながら収益を得られ、AI エージェントがその戦略に自律的にアクセスできる世界を作る」**
